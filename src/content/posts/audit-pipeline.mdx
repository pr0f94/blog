## The Problem

There are 60,000+ plugins on wordpress.org. Manual auditing doesn't scale.

I wanted a tool that could:
1. Download plugins in bulk
2. Statically analyse each one for common vulnerability patterns
3. Rank findings by likelihood of exploitability
4. Feed confirmed candidates into Burp for dynamic testing

## Architecture

```
wordpress.org API
      │
      ▼
 Downloader          downloads top N plugins by installs
      │
      ▼
 Extractor           unzips and normalises file structure
      │
      ▼
 Static Analyser     grep-based + AST parsing
      │
      ▼
 Scorer              ranks by: sink proximity, auth req, input distance
      │
      ▼
 Reporter            markdown report + JSON for further tooling
```

## The Scoring Model

Not all raw queries are equal. The scorer assigns a risk score based on:

| Factor | Weight |
|---|---|
| Input is `$_POST`/`$_GET` | +40 |
| No `$wpdb->prepare()` | +30 |
| Endpoint is `nopriv_` | +20 |
| No nonce check | +10 |
| Input passes through a sanitiser | -20 |

Anything above 70 gets flagged as HIGH priority for manual review.

## Key Code Snippet

```python
import subprocess, re

SINKS = [r'$wpdb->query', r'$wpdb->get_row', r'$wpdb->get_results']
SOURCES = [r'$_POST', r'$_GET', r'$_REQUEST']
SAFE = [r'->prepare\(', r'absint\(', r'sanitize_text_field\(']

def analyse_file(path):
    with open(path) as f:
        src = f.read()

    sink_hits   = sum(len(re.findall(p, src)) for p in SINKS)
    source_hits = sum(len(re.findall(p, src)) for p in SOURCES)
    safe_hits   = sum(len(re.findall(p, src)) for p in SAFE)

    score = (sink_hits * 30) + (source_hits * 40) - (safe_hits * 20)
    return {"file": path, "score": score, "sinks": sink_hits, "sources": source_hits}
```

## Results So Far

In the first 3 months of running the pipeline against newly released plugins:
- **847 plugins** audited
- **23 HIGH** priority candidates surfaced
- **5 confirmed** exploitable SQLi vulnerabilities
- **3 CVEs** assigned (including CVE-2024-9821)

## Get the Tool

wp-audit is open source on GitHub. Install with:

```bash
pip install wp-audit
wp-audit scan --top 100 --output report.md
```
